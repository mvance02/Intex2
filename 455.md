- Intervention plan metrics: % plans achieved/on-hold/in-progress

**API endpoints (live in `api.py`):**

- Readiness thresholds: ≥0.70 = High, 0.40–0.69 = Medium, <0.40 = Low
- Results cached 30 min in `.NET PredictController`

**Web integration:**
- `CaseloadInventory.tsx` — colored readiness badge per resident (High/Medium/Low)

## Required Notebook Sections (All 5 Pipelines)

Each notebook must contain these 6 written sections to receive full credit:

| Section | Required Content |
|---|---|
| **1. Problem Framing** | Business question, who cares, why it matters, predictive vs. explanatory justified |
| **2. Data Acquisition, Preparation & Exploration** | Load data, EDA visuals, distributions, missing values, feature engineering decisions, joins documented, reproducible pipeline |
| **3. Modeling & Feature Selection** | Model(s) built, comparison, feature selection justified, hyperparameter tuning if relevant |
| **4. Evaluation & Interpretation** | Metrics (not just R²/accuracy), train/test split, business-term interpretation, consequences of false positives/negatives |
| **5. Causal and Relationship Analysis** | Feature importance, causal claims and their limits, correlation vs. causation discussion |
| **6. Deployment Notes** | How/where model is deployed, API endpoint reference, web app integration code |

> **Critical:** Notebooks must be fully executable top-to-bottom. A TA should be able to run each notebook and reproduce results. Verify data paths are correct relative to the repo.
