### 15 — Database deployed separately from the app (2 pts)
- PostgreSQL via Supabase — completely separate from the Railway app deployment
- App data DB: Supabase project, connection string in `appsettings.json` → `ConnectionStrings:DefaultConnection`
- Identity DB: separate Supabase schema — `backend/HopeHaven.API/Migrations/AuthIdentity/`
- EF Core migrations manage schema — `backend/HopeHaven.API/Migrations/`

---

### 16 — Data validation, error handling, and code quality (1 pt)
- Client-side: `required` fields, password length check, password match check before submit
- Server-side: model validation via ASP.NET `[Required]`, `BadRequest` on mismatches
- Error display: `ErrorAlert` component + toast notifications (`ToastContext.tsx`)
- Loading states: `SkeletonLoader`, `LoadingSpinner` components
- 204 No Content handled in `apiFetch` — `frontend/src/utils/api.ts`
- Delete cascade handled in controller before EF SaveChanges — `SupportersController.cs`
